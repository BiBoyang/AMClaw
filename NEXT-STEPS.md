# NEXT-STEPS

当前这份文件只记录“接下来最值得做什么”，不重复描述已经完成的能力。当前真实状态请看 `PLAN.md`。

## 当前优先级

### 1. 文档收口

- 同步 `README.md`、`AGENTS.md`、模块级文档到当前真实能力
- 明确哪些文档是“现状”，哪些文档是“目标”

### 2. 日志与错误结构

- 把关键链路从 `println!` / `eprintln!` 逐步收敛到统一日志体系
- 让登录、任务消费、浏览器抓取、人工补录的错误更容易检索与归因

### 3. 内容处理继续深化

- 提升通用网页正文抽取质量
- 在公众号之外逐步补分类与摘要链路
- 让 pipeline 状态比现在更明确

### 4. 汇总与调度

- 为后续 `scheduler` / `reporter` 留稳定接口
- 评估日报生成与回传的最小闭环

## 当前明确暂不做

- `tokio` 全量迁移
- `sqlx` async 化
- 模式策略的大改版
- 多聊天应用接入层重构

## 下一轮建议顺序

1. 先把文档同步到当前状态，避免继续误导
2. 再统一日志与错误结构
3. 然后补通用网页抽取、分类、摘要
4. 最后再做调度、日报与更大架构迁移

## 开工标准

下一轮继续开发前，至少保证：

- 现状文档与仓库行为一致
- 真实 Bot 登录与消息链路仍可用
- 公众号浏览器抓取链路不回退
- `retry`、`状态`、`最近任务`、`待补录任务` 这些命令的行为与文档一致
