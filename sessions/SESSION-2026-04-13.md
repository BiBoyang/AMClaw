## 晚间 session

### 8. Memory 屏蔽归属校验补强

- 发现 `forget/忘记` 仅按 `memory_id` 更新，存在“知道别人的 id 就能屏蔽别人的记忆”的越权风险
- `TaskStore::suppress_memory` 改为接收 `user_id + memory_id`
- SQL 条件收紧为 `WHERE id = ? AND user_id = ? AND status = 'active'`
- 当 `UPDATE` 影响行数为 `0` 时，不再误报成功，改为返回“未找到该记忆，或无权屏蔽”
